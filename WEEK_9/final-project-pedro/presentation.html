"Was this the client's fault, or ours?"

Risk Assessment

Top risks — and how they are managed

AI Classification ErrorMEDIUM

At 92% accuracy, 8% of invoices receive a wrong recommendation. A wrong YES incorrectly charges a client; a wrong NO costs VaultGuard money.

▸ LLM-as-judge flags disagreements automatically. Human reviews only flagged cases before Oracle upload.

OpenAI DependencyMEDIUM

If OpenAI API is unavailable, the pipeline halts — delaying Oracle upload.

▸ Add fallback provider (Anthropic / Azure OpenAI). n8n retry logic already in place.

PII in Invoice PDFsHIGH → MITIGATED

Invoice PDFs contain customer names and addresses that must not be retained or sent unnecessarily to external models.

▸ SHA-256 pseudonymisation applied before every classification call. Only OCR extraction sends raw PII — documented and justified.

Invoice Format ChangeLOW

If the service provider changes the invoice layout, extraction accuracy may drop without warning.

▸ LangSmith monitors accuracy continuously and flags degradation. Re-prompt on new format.

These risks are manageable — the system is designed with human oversight at every critical point.

EU AI Act & GDPR

EU AI Act

LIMITED RISK
  • Not in Annex III high-risk categories
  • Operates in B2B billing context — not public-facing
  • Human review required before any Oracle upload
  • AI output is a recommendation, not an autonomous decision
  • Transparency: users know AI is involved in the workflow

Obligations: transparency to operators. Already documented in workflow and output files.

GDPR

MANAGED
  • Customer PII pseudonymised via SHA-256 before every classification call
  • OCR extraction sends raw PII to OpenAI — legal basis: legitimate interest
  • No customer data stored beyond the processing window
  • Same customer always gets the same pseudonym token — consistent audit trail

Gap for production: formal DPIA recommended before full deployment.

"This system can be deployed compliantly — human oversight is built in, PII is minimised at the model boundary, and the AI acts as a recommendation engine, not a decision-maker."
